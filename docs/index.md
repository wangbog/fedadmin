# FedAdmin Documentation

Welcome to the FedAdmin documentation system. This directory contains supplementary documentation for the FedAdmin federated identity management system.

## Documentation Structure

FedAdmin has two main documentation entry points with different purposes:

### 1. [README.md](../README.md) - Project Overview
**Location**: Project root directory  
**Purpose**: High-level project overview, features, and technical details  
**Content**:
- Project overview and system architecture
- Key features and technology stack
- Storage management details
- SAML metadata processing information
- Installation prerequisites
- License information

### 2. Documentation Index - User & Developer Guides
**Location**: `/docs/` directory (this file)  
**Purpose**: Detailed guides for users, developers, and operators  
**Content**:
- [User Guide](guides/user_guide.md) - How to use FedAdmin
- [Scheduled Tasks](guides/scheduled_tasks.md) - System automation and maintenance tasks
- [Development Setup](deployment/deployment_dev.md) - Development environment setup
- [Production Deployment](deployment/deployment_prod.md) - Production deployment guide
- [Technical References](reference/) - Glossary, issues, backlog, and diagrams

## Recommended Reading Order

### For New Users:
1. **Start with [README.md](../README.md)** to understand what FedAdmin is and its key features
2. **Read [User Guide](guides/user_guide.md)** to learn how to use the system
3. **Check [Known Issues](reference/issues.md)** for any current limitations
4. **Refer to [Glossary](reference/glossary.md)** when encountering unfamiliar terms

### For Developers:
1. **Read [README.md](../README.md)** for project overview
2. **Follow [Development Setup](deployment/deployment_dev.md)** to configure your environment
3. **Check [Backlog](reference/backlog.md)** for planned improvements

### For Operators:
1. **Read [README.md](../README.md)** for system overview
2. **Follow [Production Deployment](deployment/deployment_prod.md)** for installation
3. **Review [Known Issues](reference/issues.md)** and troubleshooting guides
4. **Consult [User Guide](guides/user_guide.md)** for operational workflows
5. **Review [Scheduled Tasks](guides/scheduled_tasks.md)** for system maintenance information
